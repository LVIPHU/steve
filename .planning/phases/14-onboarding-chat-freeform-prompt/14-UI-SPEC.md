| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline icon-to-text padding |
| sm | 8px | Button internal gap (icon + label), input-to-button gap |
| md | 16px | Card padding, textarea padding, input area padding |
| lg | 24px | Heading block bottom margin, card outer margin |

Exceptions:

## Color

All colors use CSS variables from `globals.css` (shadcn neutral palette, oklch).

| Role | CSS Variable | Light Value | Usage |
|------|-------------|-------------|-------|
| Dominant (60%) | --background | oklch(1 0 0) = white | Page background, full-screen canvas |
| Secondary (30%) | --card / --muted | oklch(1 0 0) / oklch(0.97 0 0) | Card surface, input background |
| Accent (10%) | --primary | oklch(0.205 0 0) = near-black | Submit button background, focus ring |
| Destructive | --destructive | oklch(0.577 0.245 27.325) = red | Toast error only |

Accent (--primary) reserved for:
- Submit button fill (primary variant)
- Loading spinner color (Loader2 inside primary button)
- Focus ring outline on Textarea and Button

Dark mode: handled automatically via `.dark` class and `@media (prefers-color-scheme: dark)` in `globals.css`. No additional dark-mode decisions needed for this phase.

## Interaction Contract

### Layout Structure

```
<div class="mx-auto max-w-2xl px-4 mt-8">
  <heading block>            ← "Tạo website mới" + subheading
  <card>                     ← rounded-lg border border-border bg-card p-6
    <textarea>               ← rows=4, resize-none, full-width
    <hint text>              ← "Ctrl+Enter để tạo website", text-xs text-muted-foreground
    <submit button>          ← w-full, size="lg", disabled when empty or loading
```

Card has no ScrollArea — no chat history to scroll. Single viewport, vertically compact.

### Textarea Behavior

- `rows={4}` minimum (approx 96px), user can expand to `rows={6}` via `resize-y` OR keep `resize-none` (D-07 spec allows either; use `resize-none` to match existing pattern)
- `placeholder`: "Tạo landing page cho khóa học tiếng Anh online, có phần giới thiệu, bảng giá và form đăng ký"
- `autoFocus` on mount — cursor lands in textarea immediately
- `disabled` when `isCreating === true`

### Submit Button

- Label (idle): `<Sparkles h-4 w-4 mr-2 /> Tạo website`
- Label (loading): `<Loader2 h-4 w-4 mr-2 animate-spin /> Đang tạo...`
- `disabled` when: `inputValue.trim() === ""` OR `isCreating === true`
- `size="lg"` — satisfies 44px touch target requirement
- `className="w-full mt-3"` — full width below textarea

### Keyboard Shortcut

- `Ctrl+Enter` (or `Cmd+Enter` on Mac): triggers submit — matches pattern in `editor-client.tsx` line ~431
- `Enter` alone: inserts newline (default textarea behavior, no override)
- Hint text below textarea: "Nhấn Ctrl+Enter để tạo website" — `text-xs text-muted-foreground mt-1`

### Loading State

- `isCreating = true`: Textarea disabled, Button shows spinner, no navigation yet
- Replace textarea area with 2–3 Skeleton lines: `<Skeleton className="h-4 w-full" />` stacked with `space-y-2`
- Skeleton shown inside card where textarea was — no separate overlay or modal
- Duration: typically < 2s (AI name extraction ~1s + POST /api/websites ~200ms)

Source: `14-CONTEXT.md` D-03, D-08, D-09

## Interaction States Summary

| State | Textarea | Button | Skeleton |
|-------|----------|--------|----------|
| Idle (empty) | enabled, autofocus | disabled | hidden |
| Idle (has text) | enabled | enabled, primary | hidden |
| Creating | disabled | disabled, spinner | visible (replaces textarea area) |
| Error (after failed create) | re-enabled | re-enabled | hidden |

Error recovery: toast.error fires, isCreating resets to false, textarea re-enabled — user can retry without page reload.
