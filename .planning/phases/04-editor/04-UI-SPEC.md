## Spacing Scale

Declared values (must be multiples of 4). Tailwind v4 used; these map to Tailwind spacing utilities directly.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, drag-handle padding, inline gaps |
| sm | 8px | Compact sidebar items, input internal padding |
| md | 16px | Sidebar section padding, form field gaps |
| lg | 24px | Sidebar panel padding, topbar padding |
| xl | 32px | Preview area horizontal padding |
| 2xl | 48px | Major layout gaps |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets on drag handles: minimum 44px tall (accessibility — WCAG 2.5.5)
- Topbar height: 48px fixed
- Sidebar width: 35% of viewport (min 320px, max 480px)
- Preview container width variants: full (Desktop), 768px (Tablet), 390px (Mobile)

## Color

CSS variables from globals.css — shadcn neutral palette, dark mode adaptive.

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | --background: oklch(1 0 0) = #ffffff | oklch(0.145 0 0) = ~#1a1a1a | Editor page background, preview canvas background |
| Secondary (30%) | --sidebar: oklch(0.985 0 0) = ~#f7f7f7 | oklch(0.205 0 0) = ~#2a2a2a | Sidebar panel, topbar surface, section list background |
| Accent (10%) | --primary: oklch(0.205 0 0) = ~#1a1a1a | oklch(0.922 0 0) = ~#ebebeb | See reserved list below |
| Destructive | --destructive: oklch(0.577 0.245 27.325) = ~#dc2626 | oklch(0.704 0.191 22.216) | Discard changes action only |
| Muted | --muted-foreground: oklch(0.556 0 0) | oklch(0.708 0 0) | Placeholder text, disabled states, drag handles |
| Border | --border: oklch(0.922 0 0) | oklch(1 0 0 / 10%) | Sidebar dividers, section list borders, input borders |

Accent (--primary) reserved for:
1. Save button (primary action in topbar)
2. Active/selected section highlight — 2px border outline on preview click-target
3. Active tab indicator in sidebar Sections / Theme tabs
4. "Regenerate section" button active state

Accent is NOT used on: section list items, form field labels, helper text, nav links, drag handles.

Preview section selection highlight: 2px solid border using `ring-2 ring-primary` — applies to the clicked section wrapper in the live preview.

## Layout Contract

### Editor Page Structure

```
┌─────────────────────────────────────────────────────┐
│ TOPBAR (48px fixed height, --sidebar bg)            │
│  ← Back  |  Website Name*  |  [Save ●]  [Preview]  │
├───────────────────────────┬─────────────────────────┤
│                           │                         │
│   PREVIEW (65%)           │   SIDEBAR (35%)         │
│                           │                         │
│   Responsive container    │   [Sections] [Theme]    │
│   (Desktop/Tablet/Mobile) │   ─────────────────     │
│                           │   Section list w/ ☰     │
│   Click section →         │   ─────────────────     │
│   ring-2 ring-primary     │   Active section form   │
│   highlight               │                         │
└───────────────────────────┴─────────────────────────┘
```

### Topbar
- Height: 48px, `bg-sidebar border-b border-border`
- Left: Back button (`← Back`) — ghost variant, navigates to `/dashboard/websites/[id]`
- Center: Website name, 20px semibold — appended with `*` when unsaved changes exist
- Right: Save button — primary variant; dot indicator `●` appears before "Save" label when unsaved changes exist
- Responsive toggle: Desktop | Tablet | Mobile — segment control, icon-only on small screens

### Sidebar
- Tab bar: "Sections" and "Theme" — active tab uses bottom border `border-b-2 border-primary`
- Section list item height: 44px (touch target), `flex items-center gap-2 px-3`
- Drag handle ☰: lucide `GripVertical` icon, color `text-muted-foreground`
- Active section in list: `bg-accent text-accent-foreground`
- Section edit form: appears below list when a section is selected, separated by `border-t border-border`

### Preview
- Preview canvas: `bg-background` (white in light mode)
- Inactive section wrapper: no visible ring
- Hovered section wrapper: `ring-1 ring-border` (cursor: pointer)
- Active/selected section wrapper: `ring-2 ring-primary` — 2px primary color outline

## Interaction Contract

### Section Click (Preview → Sidebar)
1. User clicks section in preview
2. Section wrapper receives `ring-2 ring-primary` (CSS transition: 150ms)
3. Sidebar scrolls to that section in list (scrollIntoView)
4. Section list item receives `bg-accent` state
5. Edit form expands below list

### Drag-and-Drop (dnd-kit)
- Drag handle: lucide `GripVertical` — only the handle initiates drag, not the whole row
- DragOverlay: semi-transparent clone of dragged section row (`opacity-80`)
- Drop indicator: 2px primary line between target positions
- After drop: section order updates in local state immediately; unsaved indicator activates

### Real-time Preview
- Controlled state in `EditorClient` component
- Each keypress in sidebar form → preview re-renders via `useMemo`/direct state update
- No debounce — immediate reflection (data is local, no network call)
- Save button triggers `PATCH /api/websites/[id]` with full AST

### Responsive Toggle
- Desktop: preview container `w-full` — button `aria-label="Desktop preview"`
- Tablet: preview container `max-w-[768px] mx-auto` — button `aria-label="Tablet preview"`
- Mobile: preview container `max-w-[390px] mx-auto` — button `aria-label="Mobile preview"`
- Transition: `transition-all duration-200 ease-in-out`

### Image Upload
- Field in sidebar: label "Hình ảnh" + upload button (`variant="outline"`, lucide `Upload` icon)
- Upload state: button replaced with `<Skeleton>` / spinner while uploading
- Success: `<img>` thumbnail (48x48px, `object-cover rounded`) appears; preview updates
- Error: inline error text below field: "Tải ảnh thất bại. Thử lại." (destructive color)
- No file size indicator shown — keep UI minimal

### Color Picker (Theme tab)
- Label: "Màu chủ đạo"
- Native `<input type="color">` — no library
- Width: 40px x 32px inline swatch
- Adjacent hex text input (7-char string, controlled) for manual entry
- Change applies to preview immediately via CSS variable on preview container

### Font Selector (Theme tab)
- Label: "Font chữ"
- `<Input>` with search; debounce 300ms → query `https://fonts.googleapis.com/css2?family=...`
- Dropdown list below input: up to 8 font suggestions
- On select: inject `<link rel="stylesheet">` into document head; apply `--font-family` CSS variable
- No external registry required — direct Google Fonts CSS API

### Unsaved Changes Guard
- On navigate away (back button or browser navigation): `window.onbeforeunload` + Next.js router event
- Dialog (shadcn `dialog` component):
  - Title: "Bạn có thay đổi chưa lưu"
  - Body: "Rời trang sẽ mất các thay đổi chưa được lưu."
  - Actions: "Rời trang" (destructive variant) | "Tiếp tục chỉnh sửa" (outline variant)
