    **Step 1: Create `src/lib/component-library/index.ts`**

    Export the `ComponentSnippet` interface with exactly these fields:
    ```typescript
    export interface ComponentSnippet {
      id: string;
      name: string;
      description: string;
      category: string;
      tags: string[];
      priority: number;
      domain_hints: string[];
      min_score: number;
      fallback: boolean;
      fallback_for: string[];
      html: string;
    }
    ```

    Export `selectComponents(analysis: AnalysisResult): ComponentSnippet[]` implementing this algorithm:
    1. Build candidate tag set: `new Set([analysis.type, ...analysis.sections, ...analysis.features])`
    2. Score each snippet: count of `snippet.tags` that appear in the candidate set
    3. Filter eligible: snippets where `score >= snippet.min_score`
    4. If fewer than 4 eligible, re-filter with threshold 0 (all snippets with score >= 0)
    5. Sort eligible: score desc, then domainBoost desc (snippet.domain_hints.includes(analysis.type)), then priority asc
    6. Take top 4
    7. FALLBACK PATH (separate from min_score filtering): if ALL scored items have score === 0, collect snippets where `snippet.fallback === true && snippet.fallback_for.includes(analysis.type)`. If none found for that type, fall back to `fallback_for.includes("generic")`. Return up to 4. This path reads directly from ALL_SNIPPETS, NOT from the eligible array (bypasses min_score).

    Import `ALL_SNIPPETS` from `./snippets/index`.
    Import `AnalysisResult` from `@/lib/ai-pipeline/types`.
