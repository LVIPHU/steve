## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline chip spacing |
| sm | 8px | Button inner padding (vertical), label-to-input gap |
| md | 16px | Card padding, form field spacing, section component padding |
| lg | 24px | Page section vertical gap, preview panel padding |
| xl | 32px | Dashboard page content top margin |
| 2xl | 48px | Public website section vertical rhythm |
| 3xl | 64px | Public website hero padding-top / hero padding-bottom |

Exceptions:
- Slug input + Publish button row: 8px gap (between label and button — aligns with sm token)
- Generate button: 48px min-height (touch-target floor, aligns with 2xl token; exceeds 44px accessibility minimum)

## Component Inventory

### Dashboard Detail Page (`/dashboard/websites/[id]`)

Primary focal point: Generate button (full-width, accent background, above the fold). All other interactive elements (slug input, Publish button, preview panel) are hidden until generation completes, ensuring the Generate button is the sole call to action on first view.

| Component | Source | State variants |
|-----------|--------|---------------|
| `Button` (Generate) | `src/components/ui/button.tsx` | default, loading (spinner + disabled), regenerate variant |
| `Button` (Publish) | `src/components/ui/button.tsx` | hidden until content generated; default when visible |
| `Input` (slug) | `src/components/ui/input.tsx` | hidden until content generated; editable; validated |
| `Card` (preview container) | `src/components/ui/card.tsx` | hidden until content generated; visible with section render |
| Loading spinner | inline SVG or lucide `Loader2` with `animate-spin` | active during generation only |

## Interaction States

### Generate Button Flow

| State | Visual |
|-------|--------|
| Idle (no content yet) | Button text: "Generate Website" — full width, accent background |
| Loading | Button text: "Generating…" + `Loader2` icon (animate-spin) — disabled, accent background at 70% opacity |
| Complete (first time) | Button replaced by: "Regenerate" (secondary variant) + Publish area appears |
| Regenerate loading | "Regenerating…" + spinner — same disabled pattern |
| Error | Button returns to idle; error banner shown above button |

### Slug Input + Publish Flow

- Slug input and Publish button are hidden (`hidden` class) until website.content is non-null.
- On reveal: slug input pre-filled with `seo.slug` from AI response; editable; 32-character max; lowercase-only validation.
- Slug prefix display: `your-domain.com/[username]/` shown as non-editable text left of input.
- Publish button: full width below slug input, accent background. Disabled while slug input is empty.
- Row gap between slug input label and Publish button: 8px (sm token).
- After publish: status badge on page updates to "Published"; Publish button changes to "Update Slug" (secondary).

### Preview Panel

- Hidden until generation complete.
- Renders Section components directly inline (not iframe — iframe is Phase 4).
- Top of preview: label "Preview" (12px, weight 400, muted color).
- Preview container: Card with 1px border, `--border` color, 16px padding, overflow-y scroll if content tall.
