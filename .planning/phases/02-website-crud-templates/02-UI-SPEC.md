## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, status badge horizontal padding (px-1) |
| sm | 8px | Inline spacing between card elements, gap in button groups |
| md | 16px | Card internal padding, form field gaps, grid gap on mobile |
| lg | 24px | Section vertical padding, card py (matches Card py-6 = 24px) |
| xl | 32px | Page main padding-top/bottom (py-8 = 32px) |
| 2xl | 48px | Empty state illustration margin-bottom |
| 3xl | 64px | Not used in Phase 2 |

Exceptions:
- Template grid uses `gap-4` (16px) between cards — matches md token
- Website card grid uses `gap-4` (16px) on mobile, `gap-6` (24px) on md+ breakpoint
- Hover dropdown menu icon (⋮) button: 32px touch target (size-8, matches Button size="icon")
- Inline rename input: full card-title width, no extra padding beyond input.tsx defaults

## Color

All colors are shadcn CSS variables — no raw hex values in component code. Light mode values from globals.css shown for reference.

| Role | CSS Variable | Light Mode Approx | Usage |
|------|-------------|-------------------|-------|
| Dominant (60%) | `bg-background` / `--background` | oklch(1 0 0) = white | Page background, main content area |
| Secondary (30%) | `bg-card` / `--card` | oklch(1 0 0) = white (dark: oklch(0.205)) | Website cards, template cards, form container, sidebar |
| Accent (10%) | `bg-primary` / `--primary` | oklch(0.205 0 0) = near-black | Reserved for: primary CTA button ("Tao website moi"), active tab underline border, selected template card ring, focused input ring |
| Destructive | `bg-destructive` / `--destructive` | oklch(0.577 0.245 27.325) = red | Delete confirmation button only |

Status badge colors (semantic — separate from 60/30/10 surface distribution):
- Published: `bg-green-100 text-green-700 dark:bg-green-900/30 dark:text-green-400`
- Draft: `bg-gray-100 text-gray-600 dark:bg-gray-800 dark:text-gray-400`
- Archived: `bg-orange-100 text-orange-700 dark:bg-orange-900/30 dark:text-orange-400`

Source: RESEARCH.md Pattern 8, confirmed as project decision in CONTEXT.md.

Accent reserved for (explicit list):
1. Primary CTA button fill ("Tao website moi") — `Button variant="default"`
2. Active tab indicator underline (`border-primary`)
3. Selected template card ring (`ring-2 ring-primary`)
4. Form submit button in /new page — `Button variant="default"`
5. Focused/active input ring (`focus-visible:ring-ring/50` via input.tsx) — uses `--ring` not `--accent`

NOT accent: hover states on cards, dropdown menu items, status badges, secondary actions.

Muted / secondary text: `text-muted-foreground` for descriptions, placeholders, empty state body.

## Interaction States

### WebsiteCard State Machine

```
idle
  → hover: show ⋮ button (opacity-0 → opacity-100, transition-opacity)
  → ⋮ click: open dropdown menu
    → "Rename" click: enter rename mode
      → Enter key: submit rename → API PATCH → router.refresh() → idle
      → Esc key: cancel → idle
    → "Doi trang thai" hover: show sub-menu (Draft / Published / Archived)
      → click status: API PATCH → router.refresh() → idle
    → "Xoa" click: enter confirm-delete mode
      → "Xac nhan" click: API DELETE → router.refresh() → card removed
      → "Huy" click: exit confirm-delete mode → idle
  → card body click (not ⋮): navigate to /dashboard/websites/[id]
```

### TemplateSuggestionBanner

```
hidden (default)
  → Note ID input blur with non-empty value: run suggestTemplate()
    → null result: remain hidden
    → non-null result: show banner with template name
  → X button click: hidden (dismissed for session)
  → User selects a different template manually: banner remains but has no binding effect
```

### Create Form Loading State

```
idle
  → "Tao website" button click with valid form: loading state
    → loading: button disabled + spinner, form inputs disabled
    → Server Action resolves: redirect to /dashboard/websites/[id]
    → Server Action returns error: show error message below form, return to idle
```

## Accessibility

| Pattern | Requirement |
|---------|-------------|
| Card ⋮ button | `aria-label="Mo menu"`, `aria-expanded` reflects open state |
| Dropdown menu | `role="menu"`, items `role="menuitem"`, keyboard navigation (Arrow keys, Enter, Esc) |
| Template cards | `role="radio"` within `role="radiogroup"`, `aria-checked` on selected card |
| Inline rename input | `aria-label="Doi ten website"`, `autoFocus` on mount |
| Delete confirm | Focus trapped within confirm panel while open; Esc dismisses |
| Status badges | `aria-label="Trang thai: [status]"` to supplement color cue |
| Tab UI | `role="tablist"`, `role="tab"`, `aria-selected`, `aria-controls` |
| Loading button | `aria-busy="true"` during Server Action in-flight |
| Disabled Generate button | `title` attribute provides tooltip text (no custom tooltip component needed in Phase 2) |

## Animation Contract

Using `motion/react` (project-installed). Source: RESEARCH.md Standard Stack.

| Element | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| Website card grid entrance | stagger fade-up: `initial={{ opacity: 0, y: 8 }}` → `animate={{ opacity: 1, y: 0 }}`, stagger 0.04s between cards | 200ms | ease-out |
