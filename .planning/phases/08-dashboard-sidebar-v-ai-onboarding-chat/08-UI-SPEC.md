## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, timestamp label padding |
| sm | 8px | Chat bubble inner padding (px-3 py-2), nav item gaps |
| md | 16px | Sidebar content padding, chat input padding |
| lg | 24px | Section gaps in sidebar, card padding |
| xl | 32px | Sidebar width breakpoints, page-level gaps |
| 2xl | 48px | Chat area vertical breathing room |
| 3xl | 64px | (reserved for full-page layouts) |

Exceptions:
- Sidebar fixed width: 240px (from CONTEXT.md decision — not a spacing token)
- Mobile hamburger touch target: 44px minimum (accessibility)
- Avatar size: 32px (w-8 h-8) for sidebar user area

## Color

All values are CSS variables from globals.css — automatically adapt to light/dark mode.

| Role | CSS Variable | Light Approx | Dark Approx | Usage |
|------|-------------|-------------|-------------|-------|
| Dominant (60%) | `var(--background)` | #ffffff | #1a1a1a | Page background, main content area |
| Secondary (30%) | `var(--sidebar)` / `var(--card)` | #f9f9f9 / #fff | #2a2a2a / #2a2a2a | Sidebar surface (`bg-sidebar`), card surfaces |
| Accent (10%) | `var(--sidebar-accent)` | #f5f5f5 | #3a3a3a | Active nav item background only |
| Destructive | `var(--destructive)` | oklch red | oklch red | Sign out confirmation, delete actions only |

Accent reserved for:
1. Active sidebar nav item background (`bg-accent rounded-md`)
2. Sidebar nav item hover background (`bg-accent/50`)
3. Nothing else — primary CTA buttons use `bg-primary`, not accent

Sidebar border: `border-r border-border` (from CONTEXT.md decision, confirmed via `--sidebar-border` var).

Chat bubbles:
- User message: `bg-primary text-primary-foreground` (from existing editor-client.tsx pattern)
- Bot message: `bg-muted text-foreground` (from existing editor-client.tsx pattern)
- Error message: `bg-destructive/10 text-destructive` (from existing editor-client.tsx pattern)

### Editor Page (`/dashboard/websites/[id]/edit`)

Editor keeps full-screen layout — no sidebar applied. The `(dashboard)/layout.tsx` sidebar wraps all dashboard pages EXCEPT the editor. The editor page uses its own standalone layout (already implemented in Phase 7).

Implementation note: Apply sidebar only within `/dashboard/**` routes except `/dashboard/websites/[id]/edit`. Best approach: sidebar lives in `(dashboard)/layout.tsx`, but the edit page overrides with its own layout via a nested route group or the editor page explicitly ignores the sidebar wrapper via CSS (`ml-0 w-full` override). Exact approach is at executor discretion — the contract is: edit page = full screen, no sidebar visible.

## Interaction Contract

### Chat Flow (Dashboard Onboarding)

| Step | Bot Says | User Action | State |
|------|----------|-------------|-------|
| 1 | Greeting + Question 1 (topi) | Type answer + Enter | `waiting_q1` |
| 2 | Question 2 (muc tieu) | Type answer + Enter | `waiting_q2` |
| 3 | Summary + CTA button | Click [Tao website nay!] | `creating` |
| 4 | (redirect) | — | Done |

- Bot typing indicator: show `Skeleton` (two lines, `w-32` + `w-20`, `h-4`, `rounded`) while AI call in-flight
- Send disabled while bot is responding (`isGenerating === true`)
- Enter key sends message (Shift+Enter = newline)
- CTA button appears as a full-width `Button` variant=`default` inside the chat scroll area, after summary message
- Chat resets on every page load (no persistence for onboarding chat)

### Chat History (Editor)

- Load `chat_history` from DB on editor open — prepopulate `messages` state
- Auto-save `chat_history` to DB via PATCH after each successful message exchange (500ms debounce, same pattern as auto-save in Phase 7)
- On publish: PATCH `chat_history = []` + reset `messages` state to `[]`

### Sidebar Nav Active State

- Active route detected via `usePathname()` hook (Next.js)
- `/dashboard` exact match: Dashboard item active
- `/dashboard/websites` and sub-paths: Websites item active
- Active class applied: `bg-accent rounded-md text-foreground font-medium`

### Mobile Hamburger

- Button position: fixed top-left, `p-2`, `z-50`, visible only on `< md`
- Click: open Sheet (slide from left, `w-[240px]`)
- Clicking a nav link inside Sheet: close Sheet + navigate
