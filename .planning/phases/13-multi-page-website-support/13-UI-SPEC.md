## Color

All values reference existing CSS custom properties from `globals.css` (oklch tokens). No new tokens introduced.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (oklch 1 0 0 light / oklch 0.145 0 0 dark) | Editor background, iframe surround, topbar bg |
| Secondary (30%) | `--muted` / `--secondary` (oklch 0.97 0 0 light) | Inactive page tabs, panel background, iframe border area |
| Accent (10%) | `--primary` (oklch 0.205 0 0 light / oklch 0.922 0 0 dark) | Active page tab indicator, active tab text |
| Destructive | `--destructive` (oklch 0.577 0.245 27.325) | Delete page confirmation only |

Accent reserved for:
1. Active Page Manager tab — 2px bottom border + text color shift to `--primary`
2. Add page `[+]` button hover state

Do NOT use accent on: inactive tabs, toolbar "Save" button (use `variant="outline"`), iframe border.

## Component Specifications

### Page Manager Tab Strip

Location: Above the iframe preview panel, below the topbar. Full width of the left 60% panel.

```
┌─────────────────────────────────────────────────────────────────────┐
│  [ Index ] [ About ] [ Contact ] [+]                                │ ← 36px height
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│                        iframe preview                               │
│                                                                     │
```

Tab strip specs:
- Height: 36px
- Tab padding: 8px horizontal, 8px vertical (sm token)
- Active tab: `border-bottom: 2px solid var(--primary)`, text `text-foreground font-semibold`
- Inactive tab: `text-muted-foreground`, hover → `text-foreground bg-muted/50`
- `[+]` add button: icon-only, `variant="ghost" size="sm"`, uses `Plus` icon from lucide
- Tab close `×`: appears on hover over non-index tabs only; `variant="ghost" size="icon-xs"`, uses `X` icon 12px; index tab has no close button
- Tab strip background: `bg-background`, border-bottom: `border-b border-border`
- Tab strip horizontal overflow: scroll if tabs exceed width (no wrap)

### Add Page Dialog

Trigger: Click `[+]` button in tab strip.

```
Dialog:
  Title: "Thêm trang mới"
  Body:  Label "Tên trang" + Input (placeholder: "vd: about, contact, pricing")
  Footer: [Hủy] [Tạo trang]
```

Input validation:
- Lowercase letters, numbers, hyphens only
- No spaces (hint text: "Chỉ dùng chữ thường, số, dấu gạch ngang")
- Cannot duplicate existing page name
- Cannot be empty on submit

### Delete Page Confirmation

Trigger: Click `×` on tab (non-index tabs only).

Mechanism: Use `Dialog` component (not browser `confirm()`).

```
Dialog:
  Title: "Xóa trang?"
  Body:  "Trang '[page name]' và toàn bộ nội dung sẽ bị xóa vĩnh viễn."
  Footer: [Hủy] [Xóa trang]   ← Xóa trang uses variant="destructive"
```

### Export ZIP Button

Location: Editor topbar, between "Lưu thay đổi" and "Xuất bản" buttons.

```
[ Lưu thay đổi ]  [ Tải ZIP ]  [ Xuất bản ]
```

Button specs:
- Label: "Tải ZIP"
- Variant: `variant="outline" size="sm"`
- Icon: `Download` from lucide, placed left of text
- Disabled state: disabled when website has no pages content yet
- Loading state: shows `Loader2` spinner, text becomes "Đang tải..."

## Interaction States

### Page Switch

When user clicks a tab:
1. Active tab indicator updates immediately (no async delay)
2. iframe `srcDoc` transitions to new page HTML
3. Chat panel clears visible messages and loads per-page chat history
4. No loading spinner needed for tab switch (instant local state change)

### Generate New Page

When user sends first chat message on a blank page:
1. Pipeline progress messages appear in chat (same STEP_LABELS pattern as existing)
2. iframe shows loading overlay (existing `isGenerating` pattern)
3. After generation: iframe updates with new HTML, tab strip shows page name still active

### Save Behavior

- Page content saves via 500ms debounce after generation (existing pattern, extended to `pages[pageName]`)
- Switching tabs does NOT trigger an explicit save — content is in React state and persisted on next change
- Per-page chat history saves via 500ms debounce after message changes (extended from current `chatHistory` pattern)

## Accessibility Requirements

- Page tab strip must be navigable by keyboard (arrow keys between tabs)
- Active tab must have `aria-selected="true"`
- Delete `×` button must have `aria-label="Xóa trang [name]"`
- Add page `[+]` button must have `aria-label="Thêm trang mới"`
- Dialogs must trap focus (shadcn Dialog handles this via Radix)
- Destructive button in delete dialog must not be the default focused element on open
