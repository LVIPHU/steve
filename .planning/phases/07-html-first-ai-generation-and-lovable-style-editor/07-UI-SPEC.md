## Spacing Scale

Declared values (applied via Tailwind v4 utility classes). Inherited from Phase 6 — no exceptions added.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, icon-to-label spacing, message bubble timestamp gap |
| sm | 8px | Compact element spacing, chat input button gap, badge padding |
| md | 16px | Default element spacing, panel padding, message bubble padding |
| lg | 24px | Section internal padding, topbar horizontal padding |
| xl | 32px | Layout gaps, panel separation |
| 2xl | 48px | Major layout breaks (topbar height + content area offset) |
| 3xl | 64px | Page-level vertical spacing (landing page hero padding) |

Exceptions:
- Topbar fixed height: 48px (matching Phase 4 editor topbar pattern — not on spacing scale, purely a layout dimension)
- Chat input textarea: min-height 40px, max-height 120px (auto-grows with content)
- iframe preview area: height = `calc(100vh - 48px)` (fills remaining vertical space below topbar)
- Touch targets for all interactive controls: minimum 44px height (WCAG 2.5.5)

| Body | 16px | 400 (regular) | 1.5 | Chat message content, form labels, description text |

## Visual Focal Point

**Editor primary screen:** The iframe preview (left 60%) is the dominant visual anchor. It fills the full height below the topbar. The right panel (40%) is visually subordinate — lower contrast, narrower. The user's attention is always drawn left to the live result. The right panel serves input, not display. The iframe border (`border border-border`) creates a clean separation without competing with the preview content.

**Loading state:** When AI is generating, a full-coverage overlay on the iframe (semi-transparent `--background` at 70% opacity + centered spinner + "Đang tạo..." text) blocks interaction with the stale preview. This overlay makes the generation state impossible to miss without obscuring the panel layout.

## Interaction Contracts

### Chat Panel

| Interaction | Behavior |
|-------------|---------|
| Send prompt | User clicks Send or presses Ctrl+Enter. Textarea clears. User message bubble appears immediately. Loading overlay covers iframe. |
| AI response | On success: iframe srcdoc updated, overlay removed, assistant bubble "Đã cập nhật!" appears. Auto-save triggered after 500ms debounce. |
| AI error | Overlay removed. Error bubble appears: "Không thể tạo nội dung. Vui lòng thử lại." No iframe change. |
| Empty textarea | Send button disabled. Cursor changes to not-allowed. |
| Message history | Newest message always in view (auto-scroll to bottom on new message). Older messages scrollable. |
| Auto-generate on mount | If `initialPrompt` URL param present and `htmlContent` is empty: triggers generation automatically on mount, no user action needed. |

### Code Editor Tab

| Interaction | Behavior |
|-------------|---------|
| Tab switch to Code | Textarea populated with current `htmlContent`. No code is executed. |
| User edits textarea | Content is local state only — not auto-saved. |
| Apply button | Updates iframe srcdoc with textarea value. Triggers PATCH save to DB. Toast "Đã lưu!" on success. |
| Apply while generating | Apply button disabled during active AI generation. |

### iframe Preview

| Interaction | Behavior |
|-------------|---------|
| Loading state | Full iframe overlay: `bg-background/70 backdrop-blur-sm` + centered `Loader2` icon (lucide, 24px, animated spin) + "Đang tạo..." label (14px, muted-foreground) |
| Empty state | Placeholder centered in iframe area: icon + "Mô tả website của bạn để bắt đầu" text |
| srcdoc update | Replaces attribute directly on `<iframe>`. No page reload. Instant visual update. |
| Responsive note | iframe always fills available panel width. The generated HTML must be mobile-first (enforced by AI system prompt). |

### Save / Publish (Topbar)

| Interaction | Behavior |
|-------------|---------|
| Save button | PATCH `/api/websites/${id}` with `{ html_content }`. Toast "Đã lưu!" on success, "Lưu thất bại, thử lại." on error. |
| Publish button | PATCH status to `published`. If `html_content` is empty: disabled with tooltip "Tạo nội dung trước khi xuất bản". |
| Unsaved indicator | If `html_content` has local changes not yet saved: "•" dot next to website name in topbar. |

### Auto-save

Triggered 500ms after any successful AI generation completes. Also triggered immediately on "Apply" in Code tab. No other auto-save triggers.

## Animation Contracts

Uses `motion` package (Framer Motion successor — already installed from Phase 4).

| Animation | Component | Spec |
|-----------|-----------|------|
| Chat message entry | Each new message bubble | `initial: { opacity: 0, y: 8 }` → `animate: { opacity: 1, y: 0 }`, duration 0.15s, ease "easeOut" |
| iframe loading overlay fade in | Overlay div | `initial: { opacity: 0 }` → `animate: { opacity: 1 }`, duration 0.1s |
| iframe loading overlay fade out | Overlay div | `exit: { opacity: 0 }`, duration 0.15s, wrap in `AnimatePresence` |
| Landing page hero entry | Hero section text | `initial: { opacity: 0, y: 16 }` → `animate: { opacity: 1, y: 0 }`, duration 0.4s, stagger 0.1s for headline/subtext/CTAs |
| Feature card hover | Feature cards | CSS only: `hover:shadow-md transition-shadow duration-200` — no Framer needed |
