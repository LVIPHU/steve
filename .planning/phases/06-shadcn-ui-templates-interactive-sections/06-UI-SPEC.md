## Spacing Scale

Declared values (must be multiples of 4). Applied via Tailwind v4 utility classes.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between icon and label |
| sm | 8px | Compact element spacing, badge padding, chip margin |
| md | 16px | Default element spacing, form field gap, list item padding |
| lg | 24px | Section internal padding, card body padding |
| xl | 32px | Layout gaps, sidebar section separation |
| 2xl | 48px | Major section breaks between template sections |
| 3xl | 64px | Page-level hero padding top/bottom |

Exceptions:
- Touch targets for interactive elements (checkbox tick, flashcard flip button, quiz choice): minimum 44px height to meet WCAG 2.5.5.
- Flashcard card: minimum 240px height to give adequate reading space for definition text.
- Progress bar height: 8px (fixed — not on spacing scale, purely a component dimension).

## Typography

Dashboard and editor inherit `--font-geist-sans`. Public website pages inherit the per-website `--font-family` CSS variable.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Paragraph text, form labels, section descriptions |
| Label | 14px | 500 (medium) | 1.4 | UI labels, badge text, input labels, meta text |
| Heading | 20px | 600 (semibold) | 1.25 | Section titles (e.g. "Ingredients", "Goals") |
| Display | 36px | 700 (bold) | 1.1 | Hero headline in template public pages |

Maximum 2 font weights in dashboard/editor UI: 400 and 600.
Template public pages may use 700 bold for hero display — but only in section components, not in dashboard UI.

## Color

### Dashboard / Editor UI

Uses shadcn CSS variables from globals.css. These tokens are the source of truth.

| Role | Light mode value | Dark mode value | Usage |
|------|-----------------|-----------------|-------|
| Dominant (60%) | `--background`: oklch(1 0 0) = #ffffff | oklch(0.145 0 0) ≈ #1a1a1a | Page background, layout surfaces |
| Secondary (30%) | `--card`: oklch(1 0 0); `--sidebar`: oklch(0.985 0 0) | `--card`: oklch(0.205 0 0); `--sidebar`: oklch(0.205 0 0) | Cards, sidebar, nav, editor panels |
| Accent (10%) | `--primary`: oklch(0.205 0 0) ≈ #1a1a1a | `--primary`: oklch(0.922 0 0) ≈ #eaeaea | See reserved list below |
| Destructive | `--destructive`: oklch(0.577 0.245 27.325) ≈ #dc2626 | oklch(0.704 0.191 22.216) ≈ #f87171 | Delete actions only |

Accent reserved for:
- Primary CTA buttons ("Add Section", "Save", "Publish")
- Active/selected section ring in EditorPreview
- SyncBadge syncing spinner
- Active tab indicator
- Progress bar fill color (goals checklist)
- Quiz answer correct highlight

### Public Website Templates

Each template derives its primary accent from `theme.primaryColor` (per-website CSS variable `--primary-color`). This is user-configurable in the editor ThemeTab.

Default primary colors per template (used when no user override):

| Template | Default Primary | Rationale |
|----------|----------------|-----------|
| blog | #2563eb (blue-600) | Classic editorial link color |
| portfolio | #0f172a (slate-900) | Dark, professional |
| fitness | #dc2626 (red-600) | Energy, intensity |
| cooking | #b45309 (amber-700) | Warm, food-adjacent |
| learning | #7c3aed (violet-600) | Focus, study, knowledge |

Dark mode for public pages: toggled by adding/removing `dark` class on `<html>` element. Persisted to `localStorage` key `"theme-preference"`. System preference applied on first visit via `prefers-color-scheme` media query check before first paint.

## Interaction Contracts

### Goals Section (learning template)

| Interaction | Behavior |
|-------------|---------|
| Tick a goal | Checkbox toggles, progress bar fills proportionally, state persisted to `localStorage` keyed by `{slug}-goals` |
| Progress bar | Fills from 0% to 100% using shadcn `Progress` component, animated via CSS transition 300ms ease |
| Reset | No explicit reset button — cleared by browser data clear |
| SSR state | Checkboxes render unchecked (static), client hydration applies localStorage state |

### Quiz Section (learning template)

| Interaction | Behavior |
|-------------|---------|
| Select answer | Radio button highlights selected choice, other choices remain visible |
| Submit | Reveal correct/incorrect for all answers: green bg on correct choice, red bg on chosen-wrong choice |
| Score display | "X / Y correct" shown in heading after submit |
| Retry | "Làm lại" button resets all radio selections and score display |
| Persistence | Quiz answers persisted to `localStorage` keyed by `{slug}-quiz` |
| SSR state | Renders answer choices without selection state (static), hydration applies any saved state |

### Flashcard Section (learning template)

| Interaction | Behavior |
|-------------|---------|
| Flip | Click card or "Lật thẻ" button triggers 3D flip animation (Framer Motion `rotateY` 0→180deg, 300ms) |
| Front face | Term / concept text, centered |
| Back face | Definition / answer text, centered, revealed after flip |
| Navigation | "Trước" / "Tiếp theo" buttons (or shadcn Carousel). Current card index shown as "3 / 8" |
| Keyboard | Left/Right arrow keys navigate cards; Space key flips |
| SSR state | Renders card 1 front face (static), hydration adds flip interactivity |

### Steps Section (cooking template)

| Interaction | Behavior |
|-------------|---------|
| Tick a step | Checkbox toggles, step text gets `line-through opacity-60` style |
| Image | Each step optionally shows an uploaded image (existing Supabase Storage pattern) |
| Numbering | Bold step numbers displayed as "1.", "2.", etc. — 36px bold |
| Persistence | No persistence for steps (ephemeral cooking mode) |

### Ingredients Section (cooking template)

| Interaction | Behavior |
|-------------|---------|
| Tick an ingredient | Checkbox toggles, ingredient text gets `line-through opacity-60` style |
| Layout | Two-column grid: ingredient name left, quantity right |
| Persistence | No persistence for ingredients |

### Dark Mode Toggle

| Interaction | Behavior |
|-------------|---------|
| Toggle button | Renders as `<Toggle>` (shadcn) in template header/nav area |
| State | Adds/removes `dark` class on `document.documentElement` |
| Persistence | `localStorage` key `"theme-preference"` = `"dark"` or `"light"` |
| Initial load | Script in `<head>` reads localStorage before React hydration to prevent flash |

### Add Section Flow (Editor)

| Interaction | Behavior |
|-------------|---------|
| Entry point | "Thêm section" button at bottom of SectionsTab list |
| Picker | Dialog (shadcn `Dialog`) showing allowed section types for current template |
| Type filter | Only shows types compatible with current templateId (e.g. cooking shows `ingredients`, `steps`) |
| AI generation | Clicking a type triggers `POST /api/ai/regenerate-section` with the chosen type |
| Loading state | New section row appears with `Skeleton` placeholder while AI generates |
| Insertion | New section inserted at bottom of sections list, immediately editable |

## Template Visual Identity

Each template must look visually distinct — not just different font. Redesign applies to the layout wrapper and section components.

### Blog Template

- **Layout**: Single column, max-width 720px, generous vertical whitespace
- **Sections**: Title + date meta line above hero, body text in serif, blockquote-style for hero subtext
- **Color treatment**: Minimal — mostly black/white with blue accent links
- **Cards**: White with subtle `border` and `shadow-sm`
- **Section dividers**: `<hr>` style `border-t border-border` between sections

### Portfolio Template

- **Layout**: Full-bleed sections, max-width 1200px for content areas
- **Sections**: Hero full-height with centered text, features as 3-column grid, gallery as masonry/grid
- **Color treatment**: Dark hero (#0f172a bg), white content sections
- **Cards**: Dark card variant for features items
- **CTA**: Full-width band with accent background

### Fitness Template

- **Layout**: Edge-to-edge sections with bold section dividers, max-width 1100px content
- **Sections**: Hero with background image overlay, features as icon + bold heading rows
- **Color treatment**: Red accent on all CTAs and section headings, high contrast
- **Typography**: Oswald condensed, uppercase section labels
- **Cards**: No cards — list-style rows with left red border

### Cooking Template

- **Layout**: Recipe card aesthetic, max-width 800px, warm off-white background (`#fdf8f3`)
- **Sections**: Ingredients and Steps get card-style containers with `rounded-xl shadow-sm`
- **Color treatment**: Amber accent, earthy warm tones
- **Ingredients**: Two-column responsive grid inside card
- **Steps**: Numbered with large circle badge per step
- **Gallery**: 3-column photo grid with `rounded-lg overflow-hidden`

### Learning Template

- **Layout**: App-like, max-width 900px, slate-50 background with card-based sections
- **Sections**: Each section is a distinct `Card` component with visible card border
- **Color treatment**: Violet accent on interactive elements, progress fills, CTA buttons
- **Goals**: Progress bar at top of Goals card + checklist below
- **Flashcard**: Centered card with min-height 280px, flip animation, card navigation below
- **Quiz**: Each choice as full-width `RadioGroup.Item` styled as button

## Animation Contracts

Uses Framer Motion (`motion` package — already installed in Phase 4).

| Animation | Component | Spec |
|-----------|-----------|------|
| Flashcard flip | FlashcardSection | `rotateY` 0→180deg, duration 0.3s, ease: "easeInOut". Backface hidden on both faces. |
| Flashcard card change | FlashcardSection | `x` slide: exit left (-80px opacity 0), enter right (+80px opacity 0→1), duration 0.2s |
| Goals progress bar fill | GoalsSection | CSS transition `width` 300ms ease (shadcn Progress handles this internally) |
| Quiz answer reveal | QuizSection | `scale` 1→1.02→1 on correct answer highlight, duration 0.15s |
| Step completion | StepsSection | `opacity` 1→0.6 + `textDecoration` applied via class toggle (CSS only, no Framer needed) |
| Add section appearance | SectionsTab | `initial: opacity 0, y: 8` → `animate: opacity 1, y: 0`, duration 0.2s |
