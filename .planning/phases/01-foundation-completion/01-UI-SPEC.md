## Color

Phase 1 uses the shadcn neutral palette — no chromatic accent color. Tokens are oklch-based CSS variables.

| Role | CSS Token | Light Approx | Dark Approx | Usage |
|------|-----------|-------------|-------------|-------|
| Dominant (60%) | `--background` | #ffffff | #0a0a0a | Page background |
| Secondary (30%) | `--card` | #ffffff | #1c1c1c | Auth card, dashboard card surface |
| Surface border | `--border` | #e5e5e5 | rgba(white,10%) | Card borders, dividers |
| Primary action | `--primary` | #1a1a1a | #ebebeb | Submit button background |
| Primary foreground | `--primary-foreground` | #fafafa | #1c1c1c | Text on primary button |
| Muted text | `--muted-foreground` | #737373 | #a3a3a3 | Helper text, secondary copy |
| Destructive | `--destructive` | oklch(0.577 0.245 27.325) ≈ #dc2626 | oklch(0.704 0.191 22.216) ≈ #ef4444 | Error messages, destructive actions only |

Accent reserved for: **none in Phase 1** — the `--accent` token (neutral gray) is used only for hover states on nav items; no chromatic accent is introduced.

Color scheme: neutral grayscale (no blue, no brand color). This matches the existing codebase palette. Chromatic accent colors will be introduced in Phase 4 (color/font customization).

## New Screen Spec: /onboarding

**Purpose:** Username setup for Google OAuth users who have no `profiles` record.

**Layout:** Centered card, identical to `/register` and `/login` layout pattern.

```
flex min-h-screen items-center justify-center bg-background
  motion.main (fade-in-up, duration 0.3)
    w-full max-w-sm rounded-xl border border-border bg-card p-6 shadow-sm
      h1: "Chọn tên người dùng"  (text-xl font-semibold mb-6)
      p.muted: "Tên người dùng sẽ xuất hiện trong URL website của bạn."  (text-sm text-muted-foreground mb-4)
      form (flex flex-col gap-4)
        Label + Input [username]
        helper text: "Chỉ chữ thường, số và dấu gạch ngang. Tối thiểu 3 ký tự."  (text-xs text-muted-foreground)
        error paragraph (text-sm text-destructive) — shown on submit failure only
        Button [submit] — "Lưu tên người dùng" / loading: "Đang lưu..."
```

**Animation:** Same motion pattern as existing auth pages — `initial={{ opacity: 0, y: 10 }}`, `animate={{ opacity: 1, y: 0 }}`, `transition={{ duration: 0.3 }}`.

**No back link** — this page is a required gate, not optional.

## Register Form Modification Spec

Add username field between "Tên hiển thị" and "Email":

```
div.grid.gap-2
  Label htmlFor="register-username": "Tên người dùng"
  Input
    id="register-username"
    type="text"
    placeholder="vd: nguyen-van-a"
    value={username}
    onChange set username
    required
    minLength={3}
    maxLength={30}
    pattern="[a-z0-9][a-z0-9\-]{1,28}[a-z0-9]"
  p.text-xs.text-muted-foreground: "Chỉ chữ thường, số và dấu gạch ngang. Không thể thay đổi sau này."
```

Error display: same `<p className="text-sm text-destructive">{error}</p>` pattern already in the form. No separate field-level error — single form-level error message on submit.

## Token Login Error States

The `/login` page must handle `?error` query params and display them as inline error messages above the submit button (same `text-sm text-destructive` pattern).

| Query param value | Display copy |
|-------------------|--------------|
| `invalid-token` | "Link đăng nhập không hợp lệ. Vui lòng thử lại từ ứng dụng." |
| `expired-token` | "Link đăng nhập đã hết hạn (5 phút). Vui lòng tạo link mới từ ứng dụng." |

Implementation: read `searchParams.get('error')` in the page and render the message. No new component needed.
