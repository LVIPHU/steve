**3. Add `databaseHooks.user.create.after`:**
```typescript
databaseHooks: {
  user: {
    create: {
      after: async (user) => {
        // Only insert profile if username was provided (email/password flow)
        // Google OAuth users won't have username yet — they go through /onboarding
        const username = (user as Record<string, unknown>).username as string | undefined;
        if (!username) return;

        const RESERVED_USERNAMES = [
          "dashboard", "editor", "api", "login", "register",
          "settings", "pricing", "about", "admin",
        ];
        const USERNAME_REGEX = /^[a-z0-9][a-z0-9-]{1,28}[a-z0-9]$/;

        // Validation (defensive — form also validates, but hook is authoritative)
        if (!USERNAME_REGEX.test(username)) return;
        if (RESERVED_USERNAMES.includes(username.toLowerCase())) return;

        await db.insert(profiles).values({
          id: user.id,
          username: username.toLowerCase(),
          plan: "free",
          createdAt: new Date(),
          updatedAt: new Date(),
        });
      },
    },
  },
},
```

**Important notes for executor:**
- Keep ALL existing config (drizzleAdapter, emailAndPassword, socialProviders, session, experimental.joins).
- Use `(user as Record<string, unknown>).username` cast because TypeScript doesn't know about additionalFields values at hook time — this is a known limitation documented in RESEARCH.md Pitfall 2.
- If the DB insert fails (e.g., duplicate username), swallow the error silently here — the user record is already committed. The form-level uniqueness check (Plan 02) should prevent this from reaching the hook.
- The `profiles` and `db` imports are already in `src/lib/auth.ts` — verify before adding duplicate imports.

Replace the current implementation with:

```typescript
import { getSessionCookie } from "better-auth/cookies";
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export default function proxy(request: NextRequest) {
  const sessionCookie = getSessionCookie(request);
  const { pathname } = request.nextUrl;

  if (
    !sessionCookie &&
    (pathname.startsWith("/dashboard") ||
      pathname.startsWith("/editor") ||
      pathname.startsWith("/onboarding"))
  ) {
    return NextResponse.redirect(new URL("/login", request.url));
  }

  return NextResponse.next();
}

export const config = {
  matcher: ["/dashboard/:path*", "/editor/:path*", "/onboarding"],
};
```

This consolidates the two separate `if` blocks in the current proxy.ts into one combined check. The logic is identical — just cleaner and correctly covers `/onboarding`.

No edge runtime config — proxy.ts runs on Node.js runtime only (per RESEARCH.md State of the Art section).
