**Step 1: Add Umami script to `src/app/(public)/[username]/[slug]/page.tsx`**

Inside the `PublicWebsitePage` component's return JSX, add the conditional script tag AFTER the `<link rel="stylesheet">` tag and BEFORE the theme wrapper `<div>`:

```tsx
return (
  <>
    <link rel="stylesheet" href={fontUrl} />
    {process.env.NEXT_PUBLIC_UMAMI_URL && process.env.NEXT_PUBLIC_UMAMI_WEBSITE_ID && (
      <script
        defer
        src={`${process.env.NEXT_PUBLIC_UMAMI_URL}/script.js`}
        data-website-id={process.env.NEXT_PUBLIC_UMAMI_WEBSITE_ID}
      />
    )}
    <div
      style={
        {
          "--primary-color": ast.theme?.primaryColor,
          "--font-family": fontFamily,
          fontFamily: `"${fontFamily}", sans-serif`,
        } as React.CSSProperties
      }
    >
      <TemplateRenderer templateId={website.templateId ?? "blog"} ast={ast} />
    </div>
  </>
);
```

Key points:
- Both `NEXT_PUBLIC_UMAMI_URL` AND `NEXT_PUBLIC_UMAMI_WEBSITE_ID` must be truthy for the script to render
- `defer` attribute ensures script loads after DOM parse
- `data-website-id` is a valid HTML data attribute — no React warnings
- Script is ONLY in the public page — dashboard pages are untouched
