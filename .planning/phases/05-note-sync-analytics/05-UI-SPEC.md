## Interaction Contract

### Sync badge on WebsiteCard

- Badge renders below `StatusBadge` in `CardContent` when `syncStatus !== 'idle'`
- Gap between `StatusBadge` and `SyncBadge`: 4px (`gap-1`)
- `syncStatus === 'syncing'`: badge shows animated pulse (Tailwind `animate-pulse`)
- `syncStatus === 'synced'`: timestamp formatted as `HH:MM` using `toLocaleTimeString('vi-VN', { hour: '2-digit', minute: '2-digit' })`
- `syncStatus === 'sync_failed'`: badge is red, persistent, no dismiss mechanism
- Badge does NOT have a click action — display only

### 30-second polling

- `setInterval` fires `router.refresh()` every 30 000ms
- Polling starts on mount, clears on unmount
- No loading indicator during poll — the refresh is silent
- Only active at `/dashboard/websites` route

### Umami script injection

- Renders only when `process.env.NEXT_PUBLIC_UMAMI_URL` is truthy
- Script tag: `<script defer src="{UMAMI_URL}/script.js" data-website-id="{WEBSITE_ID}" />`
- Placed at the bottom of the public page body, outside any section wrapper
- No visible UI element — purely a `<script>` tag in the DOM
